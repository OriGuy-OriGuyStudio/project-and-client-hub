# Quote System v2.1 , Product-Adaptive Content Model (design)

> Extends the v2 admin builder (spec 2026-07-15). Adds: per-item include/optional
> classification, per-product content + legal defaults, per-product maintenance
> tiers, an "interactive tools" upsell layer, an automation classification guide,
> and updated automation pricing. Built on n8n (self-hosted), not Zapier/Make.

**Status:** awaiting Ori review, then writing-plans.

---

## 1. Goal & context

The v2 builder treats all content (differentiators, phases, bonuses, FAQ, legal,
maintenance) as ONE global set, and forces every scope item to be "included". Ori's
real workflow needs two things the current model lacks:

1. **Any feature can be either "included in the price" or an "optional upsell"**, and
   this varies per quote (a store's payment integration is included; its product
   reviews might be optional).
2. **Content must adapt to the product type** , the FAQ, bonuses, differentiators,
   phases, terms, maintenance and even the legal clauses of an automation quote are
   different from a website quote.

Plus: automation pricing was too low, maintenance must be product-specific with clear
per-package deliverables, the full legal set must match Ori's real 14 clauses + 5
additions, and a new high-value upsell layer (interactive lead-gen tools).

Website pricing is NOT changed , it was already refined.

---

## 2. Include / Optional per scope item

**Model:** `ScopeItem` gains `optional: boolean` (default `false` = included).

- **Included** (`optional=false`) , part of the base anchor and the final price. Shown
  to the client as "what's included" (value stacking, optionally with breakdown).
- **Optional** (`optional=true`) , NOT in the base anchor. Becomes a client-selectable
  add-on (priced on top when chosen), exactly like a catalog upsell.

**Pricing engine impact (`quote-pricing.ts` / `quote-v2.ts`):**
- `anchorValue` sums only scope items where `!optional`.
- `quoteTotals`: the pool of client-selectable extras = `optional` scope items +
  `content.upsells`. The 3 price options are still computed from the included-only
  anchor. Selected optional items + selected upsells add on top (same math as upsells
  today).
- Unit tests updated: anchor excludes optional; totals add optional-selected correctly.

**Builder UI:** each selected scope row gets a small segmented toggle **כלול במחיר /
תוספת אופציונלית**. Default כלול. Locked when signed.

This unifies the catalog: one list of pages/features/modules, classified per quote.
The separate upsell catalog stays for cross-product add-ons (interactive tools,
automation-into-website).

## 3. Per-product content + legal defaults

**Change:** `quote_defaults` goes from ONE global row to **one row per type**
(`website` / `system` / `automation`) via a new `type` column (PK-ish uniqueness on
`type`). Each row holds that type's own: differentiators, phases, bonuses, next_steps,
faq, legal (string[]), payment, testimonial, validity_days, and a maintenance default
(offer + which tiers).

- `useQuoteDefaultsV2()` reads the row for a given type.
- `newQuoteContentFromDefaults(type, defaults)` seeds a new quote from its type row.
- **Defaults page** (`QuoteDefaultsV2`) gets a **type switcher** (tabs אתר/מערכת/
  אוטומציה); each tab edits that type's defaults + legal + maintenance tiers + the
  upsell catalog (upsells stay universal; see §6).

Migration seeds sensible starting content per type (esp. legal, see §8).

## 4. Automation: packages, guide, n8n

Automation build is priced as **base setup + per-automation tiers** (catalog
`kind='automation'`). Each catalog row carries a `description` = the deliverables.

**Packages (ex-VAT), platform = n8n self-hosted:**

| Package | Price | What the client gets (internal checklist) |
|---|---|---|
| בסיס הקמה ושילוב | ₪3,000 | אפיון התהליך + מיפוי זרימה, הקמת סביבת n8n + חיבור המערכות (API/OAuth/webhooks), אוטומציה אחת עובדת, בדיקות end-to-end + טיפול שגיאות בסיסי, הדרכה + מסירה + 14 יום תיקונים |
| אוטומציה פשוטה | ₪2,000 | טריגר אחד, 1-2 מערכות, זרימה לינארית, בלי תנאים |
| אוטומציה בינונית | ₪4,000 | 2-3 מערכות, תנאים/הסתעפויות, מיפוי ועיבוד נתונים, טריגר לפי סטטוס |
| אוטומציה מורכבת | ₪7,000 | 3+ מערכות, סנכרון דו-כיווני, לוגיקה מתקדמת, טיפול שגיאות + retry + לוגים, אופציונלי רכיב AI |

Automation type **floor** raised to **₪3,500** (from ₪2,500) in `quote_type_multipliers`.

**Client-facing copy** for each package is benefit-worded (not the internal checklist).
Example: "אוטומציה מורכבת" client copy = "חיבור חכם בין כל המערכות שלך, שמדבר בשני
הכיוונים ומעדכן את עצמו , בלי עבודה ידנית ובלי לידים שנופלים."

**n8n sales point** (surfaced in automation content/copy): self-hosted = no per-task
SaaS fees, no volume ceiling, runs on dedicated infra , only hosting cost.

**Automation classification guide (in-app):** a collapsible info section shown in the
builder when type=automation, with the 4-level criteria (systems / directionality /
logic / error-handling) + one example each. Static content component `AutomationGuide`
(no schema). Content = the guide Ori approved:
- Base (always): setup + 1 automation.
- Simple: 1 trigger, 1-2 systems, linear.
- Medium: 2-3 systems, conditions, data mapping, status trigger.
- Complex: 3+ systems, bi-directional sync, advanced logic, error handling + retry + logs.
- Worked example: GHL+Workiz (Google/Meta Ads → GHL; GHL lead won → Workiz job; Workiz
  done → GHL status) = **complex**.

## 5. Maintenance tiers per product

**New table `quote_maintenance_tiers`** (admin-curated, per type):
`id, type ('website'|'system'|'automation'), key, name, price (numeric, monthly ex-VAT),
description (client-facing), recommended boolean, sort int`. RLS: admin ALL.

**Seed (ex-VAT / month):**

| Type | Tier | Price | Includes |
|---|---|---|---|
| אתר | Core | ₪350 | אירוח, גיבוי, אבטחה, עדכונים, תיקונים קטנים |
| אתר | Pro | ₪850 | + עריכות תוכן, ביצועים, פיצ'רים קטנים, עדיפות |
| אתר | Ultra VIP | ₪1,800 | + שעות חודשיות, SEO, אנליטיקס, SLA מהיר |
| אוטומציה | Care | ₪500 | ניטור + התראת כשל, תיקון תקלות, עדכון בשינויי API צד ג', עד 2 שינויים קטנים/חודש |
| אוטומציה | Managed | ₪1,400 | + אופטימיזציה, אוטומציה קטנה חדשה/חודש (או שעות), סקירה ודוח חודשי |
| מערכת | Core | ₪700 | אירוח, גיבוי, אבטחה, עדכוני תלויות, ניטור זמינות, תיקוני באגים |
| מערכת | Pro | ₪1,800 | + שעות פיתוח חודשיות, פיצ'רים קטנים, ביצועים, עדיפות |
| מערכת | Ultra VIP | ₪3,500 | + SLA מהיר, שעות מובטחות, ליווי צמוד, דוחות |

**Content shape change:** `content.maintenance` becomes
`{ offer: boolean, tiers: [{ key, name, price, description, recommended }] }` , a
**snapshot** of the chosen tiers (so later catalog edits do not change existing quotes,
same rule as upsells). `quoteTotals.monthlyFor` reads from the snapshot.

**Maintenance editor** (in the builder's "תוספות"/"תנאים" tab): pick which of the
type's tiers to offer; the client later selects one on the client page (Plan 3).
Defaults page (per type) curates the tier list.

## 6. Interactive Tools , new upsell layer

Ori's strongest differentiated upsell (proven DNA: Zamiron wizard + his own AI tools).
Repositions "website builder" → "digital-asset builder". Productize the engine once,
customize content per client.

**Catalog upsell rows** (`kind='upsell'`, `type=null` = universal, so available to any
quote type). Seed 3 tiers:

| Tier | What | Price (ex-VAT) |
|---|---|---|
| Quiz / בוחן התאמה | 3-6 שאלות ← תוצאה מותאמת + לכידת ליד | ₪3,000 |
| מחשבון / קונפיגורטור | לוגיקה + תמחור דינמי + תוצאה | ₪5,500 |
| כלי אינטראקטיבי מורכב | רב-שלבי, מיפוי מותאם, חיבור n8n/CRM, דשבורד לידים | ₪10,000 |

Natural attaches: lead-routing automation (n8n) + optimization retainer.

**Expectation framing (client copy):** "ממיר יותר מבקרים ללידים איכותיים" , converts
traffic, does not generate it; also qualifies/segments leads. IP: the engine stays
Ori's (license to client) , covered by the new IP legal clause (§8).

No schema work , these are ordinary upsell rows. Ori curates them in the defaults page.

## 7. Upsells model (recap)

Two sources of client-selectable extras, both snapshot-copied into the quote:
1. **Optional scope items** (§2) , any page/feature/module marked אופציונלי.
2. **Upsell catalog** (`kind='upsell'`) , cross-product add-ons: existing 3 (רב-לשוני,
   דיוור, תיאום פגישות) + interactive tools (§6) + "אוטומציה נלווית לאתר" cross-sell.

Both render as pick-lists on the client page (Plan 3) and add on top of the chosen
final price.

## 8. Legal , full set + per-product variants

**Ori's 14 approved clauses** (website baseline, verbatim from his app):
1. ביצוע על הצד הטוב ביותר לפי שיקול דעת מקצועי.
2. תוספי פרימיום רשומים על הסטודיו; אחרי שנה ייתכן רישיון עצמאי ללקוח.
3. אין אחריות לסטיות צבע במסכים/הדפסה.
4. פרויקט במסגרת וורדפרס + אלמנטור; מערכות נוספות מעבר למוסכם , בנפרד (הפלטפורמה תיקבע אחרי אפיון).
5. זכות הצגה בפורטפוליו/רשתות/מדיה + קרדיט בתחתית האתר.
6. "סבב תיקונים" = מסמך מרוכז; תיקונים מעבר לסבבים המוסכמים , ₪450/שעה.
7. העלאה לאוויר רק אחרי תשלום מלא.
8. הפסקת פרויקט מצד הלקוח , תשלומים כדמי ביטול/פיצוי, בלי זכות שימוש בחומרים.
9. לקוח שלא מספק תכנים עד הסיום , עולה עם תכני דמו, החיוב לא תלוי בסטטוס התכנים.
10. יתרת התשלום עם סיום העבודה המובטחת, ללא תלות בחומרי הלקוח; תינתן הדרכה להזנה.
11. אחרי סיום , האתר עובר לשרת הלקוח; אי-מסירת גישה עלולה להוביל למחיקה משרתי הסטודיו.
12. אחריות נגישות/זכויות יוצרים/תוכן על הלקוח; הסטודיו יתקין תוסף נגישות חינמי ויסייע בניסוח מדיניות.
13. אישור ההסכם מבטל הסכמות/מסמכים קודמים; זה ההסכם המחייב היחיד.
14. המחיר בש"ח לפני מע"מ, תוקף שבוע; אישור הלקוח = אישור רשמי בכתב.

**5 approved additions** (Ori's voice):
15. **קניין רוחני:** עם תשלום מלא זכויות השימוש בתוצר עוברות ללקוח; רכיבי תשתית/קוד שפותחו ע"י הסטודיו נשארים בבעלותו עם רישיון שימוש ללקוח.
16. **אחריות ובאגים:** 30 יום מהעלייה לאוויר לתיקון תקלות שמקורן בעבודת הסטודיו, ללא עלות; תקלות משינויי לקוח/צד ג' , בתשלום.
17. **תחזוקה:** תחזוקה שוטפת אינה כלולה במחיר, אלא אם נרכשה חבילה בנפרד.
18. **הגבלת אחריות:** אחריות הסטודיו מוגבלת לסכום ששולם; אין אחריות לנזקים עקיפים/אובדן רווחים.
19. **דין וסמכות שיפוט:** דיני מדינת ישראל, סמכות לבתי המשפט המוסמכים.

(Optional, not seeded unless Ori asks: סודיות, כוח עליון.)

**Per-product legal variants** (stored in each type's `quote_defaults.legal`):
- **אתר (website):** all of 1-19.
- **מערכת (system):** drop website-only (4 WordPress/Elementor phrasing → generic "פלטפורמה/סטאק שייקבע", 7 "העלאה לאוויר" → "העלאה לפרודקשן", 9 "תכני דמו", 11 "שרת הלקוח" adjusted); keep payment, IP, warranty, maintenance, cancellation, liability, jurisdiction, validity, portfolio.
- **אוטומציה (automation):** drop all website/site phrasing; core = ביצוע מקצועי, תשלום/ביטול, IP, אחריות ובאגים (על תקלות שמקורן בבנייה; שינויי API צד ג' , בתשלום/תחזוקה), תחזוקה לא כלולה, הגבלת אחריות, דין, תוקף; add automation-specific: "עלויות פלטפורמה/אחסון (n8n) על הלקוח", "תלות ב-API של מערכות צד ג'; שינוי מצידן עלול לחייב התאמה בתשלום/במסגרת תחזוקה".

Exact per-type legal strings are drafted in the migration seed (no placeholders).

## 10. Data / schema changes

2. `quote_defaults` , add `type` column; migrate the single row to 3 per-type rows;
   seed per-type content + per-type legal.

   recommended, sort)` + RLS admin ALL + seed (§5).
4. `content.maintenance` shape → `{ offer, tiers: [snapshot] }`.
5. `quote_catalog` , update automation rows (prices + descriptions); add 3 interactive-
   tools upsell rows; (optional) add "אוטומציה נלווית לאתר" upsell row.

All migrations applied to the **branch** DB (`dbchappsqcsixxecxzqv`) only, per workflow.
