# Quote v2.1-C , Per-product content & legal defaults (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make quote content + legal adapt to the product type: `quote_defaults` becomes one row per type (website/system/automation), each seeding its own differentiators/phases/bonuses/faq/legal + defaults; the defaults page edits per type.

**Architecture:** Add `type` to `quote_defaults`, migrate the single row to three, seed per-type content and the full legal set (Ori's 14 + 5 additions, with per-product legal variants). Hooks + defaults page become type-aware. New quotes seed from their type's row.

**Tech Stack:** Supabase (branch `dbchappsqcsixxecxzqv`), React + TS, TanStack Query.
