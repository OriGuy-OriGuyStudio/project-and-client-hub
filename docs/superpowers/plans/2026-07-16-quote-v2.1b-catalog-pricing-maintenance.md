# Quote v2.1-B , Catalog, pricing & per-product maintenance (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Update automation pricing + package descriptions, add the interactive-tools upsell layer, add per-product maintenance tiers (curated), and an in-app automation classification guide. Platform = n8n.

**Architecture:** DB seed/migrations on the branch for catalog + a new `quote_maintenance_tiers` table; `content.maintenance` becomes a snapshot of chosen tiers; hooks + a maintenance editor + a static `AutomationGuide` section. Website pricing untouched.

**Tech Stack:** Supabase (branch `dbchappsqcsixxecxzqv`), React + TS, TanStack Query.

## Global Constraints
- Branch `staging`; no push; NO "Co-Authored-By"; NO em-dashes ("—"); Hebrew RTL; semantic tokens not hex; platform copy says **n8n (self-hosted)** never Zapier/Make; `npm run build` + `npm test` green.
- Migrations are WRITTEN as files by implementers and APPLIED by the controller via Supabase MCP (branch only). Snapshot rule: choosing a tier/upsell copies its values into `content` so later catalog edits never change existing quotes.
